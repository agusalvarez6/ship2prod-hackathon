# Vapi

**Prize:**
Best Use of Vapi, 2 winners:

* Best voice AI agent: **$500 Vapi credits + AirPods for each team member**
* Most commercially viable voice agent: **$1,500 Vapi credits** ([Ship to Prod][1])

**Links:**

* Docs: [https://docs.vapi.ai/quickstart/introduction](https://docs.vapi.ai/quickstart/introduction)
* Dashboard: [https://dashboard.vapi.ai/](https://dashboard.vapi.ai/)

**Short description:**
Voice infrastructure for AI agents.

**What we use it for:**

* Let the user ask for a meeting brief by voice
* Read the 60-second summary out loud
* Let the user ask follow-up questions
* Turn the product from “dashboard” into a voice agent

# WunderGraph

**Prize:**
Best Use of WunderGraph, 3 winners:

* 1st: **$2,000**
* 2nd: **$1,000**
* 3rd: **$500** ([Ship to Prod][1])

**Links:**

* Cosmo docs: [https://cosmo-docs.wundergraph.com/overview](https://cosmo-docs.wundergraph.com/overview)
* ConnectRPC tutorial: [https://github.com/wundergraph/connectrpc-tutorial](https://github.com/wundergraph/connectrpc-tutorial)
* Hub docs: [https://hub-docs.wundergraph.com/introduction](https://hub-docs.wundergraph.com/introduction)

**Short description:**
API orchestration layer that connects services.

**What we use it for:**

* Expose one clean API to the frontend
* Connect Calendar, Notion, TinyFish, Redis, InsForge, and Vapi
* Keep the frontend from calling every service directly
* Provide endpoints like `listMeetings`, `generateBriefing`, and `askBriefingQuestion`
